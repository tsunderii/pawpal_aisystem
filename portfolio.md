## What this project says about me as an AI engineer

This project demonstrates that I build AI systems I can actually trust. I designed PawPal+ with a deliberate separation between deterministic scheduling logic and AI-generated reasoning — because I know that reliable systems don't outsource correctness to a language model. When the AI produced a weighted scoring formula, I traced through edge cases myself, wrote targeted tests, and confirmed the behavior before accepting it. When it introduced a stale-data bug in the Streamlit session state, I caught it by running the app end-to-end — not by assuming the code was right. I know how to use AI as a force multiplier without losing control of what the system actually does. That's the kind of engineer I am: someone who ships things that work, and knows *why* they work.
